# smoke-notlar-003812 — Ürün Gereksinimleri Dokümanı (PRD)

## 1. Genel Bakış

**Proje Adı:** smoke-notlar-003812  
**Platform:** Web (React 18 + Vite + TypeScript)  
**Tür:** Tek sayfa not kartları uygulaması (SPA)

Bu proje, kullanıcıların hızlıca not kartları oluşturmasını, düzenlemesini, tamamlandı olarak işaretlemesini ve silmesini sağlayan minimal bir not yönetim aracıdır. Tüm veriler tarayıcıda localStorage ile kalıcı olarak saklanır — sunucuya ihtiyaç yoktur. Türkçe arayüz ile geniş kitlelere hitap eder.

**Hedef Kitle:** Günlük görevlerini organize etmek isteyen bireysel kullanıcılar, öğrenciler, freelancerlar.

---

## 2. Hedefler

1. **Hızlı not ekleme** — Başlık ve opsiyonel açıklama ile saniyeler içinde kart oluşturma.
2. **Tamamlandı işaretleme** — Kartları "yapıldı" olarak işaretleyerek görsel ayrım sağlama.
3. **Not silme** — Gereksiz kartları tek tıkla silme (onay gerektirmez).
4. **localStorage kalıcılığı** — Sayfa yenilense bile tüm veriler korunur.
5. **Responsive tasarım** — Mobil (320px+) ve desktop (1920px+) ekranlarda düzgün çalışır.
6. **Erişilebilir UI** — Klavye navigasyonu ve WCAG 2.1 AA uyumlu.

## 4. Fonksiyonel Gereksinimler

### 4.1 Not Kartı Oluşturma

- **Zorunlu alan:** Başlık (title), en az 1 karakter, maks 100 karakter.
- **Opsiyonel alan:** Açıklama (description), maks 500 karakter.
- **Oluşturulma tarihi:** Otomatik, `new Date().toISOString()` formatında saklanır.
- **ID:** `crypto.randomUUID()` ile benzersiz atanır.
- **Varsayılan durum:** `completed: false`
- **Hata mesajları:**
  - Boş başlık: "Başlık alanı boş bırakılamaz."
  - Başlık > 100 karakter: "Başlık en fazla 100 karakter olabilir."
- **Başarı davranışı:** Yeni kart listenin en üstüne eklenir, form sıfırlanır, input'a odaklanılır.

### 4.2 Not Kartı Tamamlandı İşaretleme

- Her kartın solunda bir onay kutusu (checkbox) bulunur.
- Tıklandığında `completed` alanı tersine çevrilir (true ↔ false).
- Tamamlanmış kartlar: başlık üstü çizili (text-decoration: line-through), opacity 0.6.
- Tamamlanmamış kartlar: normal görünüm.
- **Hata mesajı yok** — toggle anlık ve güvenli.

### 4.3 Not Kartı Silme

- Her kartın sağ üstünde bir "Sil" butonu (çöp kutusu ikonu) bulunur.
- Tıklandığında kart hemen silinir (onay dialogu YOK).
- **Hata mesajı:** Silinemezse "Not silinemedi, lütfen tekrar deneyin."
- **Başarı davranışı:** Kart animasyonla (fade-out, 200ms) listeden çıkar.

### 4.4 Not Listesi Görüntüleme

- Tüm notlar kronolojik sırada (en yeni üstte) listelenir.
- Boş durum: Ortada büyük bir ikon ve mesaj: "Henüz not eklemediniz. Yukarıdaki formu kullanarak ilk notunuzu ekleyin."
- Liste performansı: 500 kart kapasitesi hedeflenir (>500 için virtual scroll gerekirse eklenir, başlangıçta gerekmez).

### 4.5 localStorage Kalıcılığı

- Her kart değişikliğinde (ekle/sil/toggle) tüm liste `localStorage.setItem('notes', JSON.stringify(notes))` ile kaydedilir.
- Sayfa yüklendiğinde `localStorage.getItem('notes')` ile liste geri yüklenir.
- Veri bozuksa veya JSON parse hatası alınırsa: boş liste ile başla, konsola hata logla.
- Storage key: `smoke-notlar-003812-notes`

### 4.6 Filtreleme / Arama

- Liste üstünde bir arama input'u bulunur: "Notları ara..."
- Yazıldığında anlık filtreleme (debounce yok, her keystroke'ta).
- Başlık ve açıklama alanlarında case-insensitive substring eşleşme.
- Eşleşme yoksa: "Sonuç bulunamadı." mesajı.
- Arama temizlendiğinde tüm liste görünür.

## 5. Veri Modeli

### Entity: Note

```typescript
interface Note {
  id:       string;      // crypto.randomUUID()
  title:    string;      // 1-100 karakter
  description: string;   // 0-500 karakter, opsiyonel
  completed: boolean;    // varsayılan: false
  createdAt: string;     // ISO 8601 timestamp
}
```

### localStorage Schema

```
Key:   smoke-notlar-003812-notes
Value: JSON array of Note objects
```

## 7. Non-Functional Requirements

### 7.1 Performans

- **İlk yükleme:** < 1.5 saniye (Vite hızlı build hedefi)
- **TTI (Time to Interactive):** < 2 saniye
- **Bundle budget:** Ana bundle < 150KB gzipped
- **State güncellemesi:** Her işlem < 16ms (60fps korunur)

### 7.2 Erişilebilirlik (WCAG 2.1 AA)

- Tüm interactive elementler `tabindex` ile klavye erişiminde
- ARIA labels: form `aria-label`, buton `aria-label`, liste `role="list"`
- Focus state: `outline: 2px solid #2563EB; outline-offset: 2px`
- Kontrast: Text (#0F172A) on Background (#F8FAFC) = 16.7:1 (> 4.5:1 ✓)
- Checkbox ve buton touch target: minimum 44x44px
- Ekran okuyucu uyumlu: Sarmalayıcı div'ler `role="listitem"` vs.

### 7.3 Tarayıcı Desteği

- **Desktop:** Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **Mobil:** iOS Safari 14+, Android Chrome 90+
- **Responsive breakpoints:**
  - Mobile: 320px – 639px (tek kolon, küçük padding)
  - Tablet: 640px – 1023px
  - Desktop: 1024px+
